
Allgemeine Programmierregeln:
1. Frontend und Backend Code wird in separate Ordner angelegt.
2. Unterordner werden für spezifische Funktionen erstellt, um eine bessere Organisation zu Gewährleisten
3. Dateinamen müssen Konsistent sein
4. Versionsverwaltung mit GIT
5. Code wird klar und prägnant kommentiert
6. Beste Practices mit PEP8
7. Funktionen sollten nicht länger als 20 Zeilen sein
8. Beschreibende Namen verwenden um lesbarkeit des Codes zu verbessern
9. Keine Duplikate
10. Klassenorientiertes Programmieren
